Приложение API Explorer предназначено для тестирования API приложений, разработанных на фреймворке Webasyst и установленных в данной установке. Сторонние сервисы приложение смотреть пока не умеет. Также приложение пока умеет смотреть только API, опубликованное в рамках стандартной экосистемы фреймворка - реализованное через наследование класса waAPIMethod и работющее через /api.php

Список доступных методов API приложение собирает автоматически, анализируя содержимое папки api каждого доступного текущему юзеру приложения. При наличии в приложении файла с описанияем методов API в формате Open API (Swagger), применяется данное описание. Файл с описанием Open API на данный момент ожидается строго по пути api/swagger/v1.yaml в папке приложения. При отсутствии описания состав методов API определяется через reflection всех классов PHP в папке api.

Процесс первичного получения полного списка методов может занимать некоторое время при открытии приложения. В целях оптимизации полученный список кешируется как в браузере, так и на сервере. Поэтому, при возникновении необходимости обновить список (например, был добавлен новый метод) требуется очищение кеша фреймворка на сервере и обновление страницы приложения в браузере.

Пользователь, не являющийся администратором системы, может выполнять вызовы API только от своего лица. Пользователь с правами администратора видит слева список всех пользователей и может выбирать персонажа, от лица которого выполнять вызовы.

В реальных условиях вне интерфейса API Explorer для обращения к API необходимо получить токен. Подробнее об авторизации и подключении API. Для упрощения работы API Explorer сам автоматически создает токен с доступом ко всем приложениям и правами выбранного пользователя при клике на имени пользователя в левом сайдбаре. Созданный в API Explorer токен можно удалить - на экране работы с любым методом API есть пункт Token.
